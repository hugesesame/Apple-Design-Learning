Core Principles

3つの核心

Appleがデザインで守り続ける原則。
すべての判断はここに帰結する。

01
アクセントカラーは1色だけ。ブルー（#0066cc）以外でCTAを着色しない。
02
UIに影は使わない。奥行きはカラー変化・ヘアライン・Frosted Glassで表現する。
03
余白が価値を作る。80pxの台座が、製品を美術品に変える。

Curriculum

12のモジュール

哲学から実装まで。
Appleのデザイン言語を体系的に解剖する。

Module 01
Appleデザイン哲学の基礎
Less, but better — 削ることがAppleのデザイン行為だ
Module 02
カラーシステム構築
セマンティックカラー — 値ではなく意図に名前を付ける
Module 03
タイポグラフィ体系
17pxは設計値。letter-spacingが文字に命を吹き込む
Module 04
スペーシングとグリッド
80pxの台座 — 余白の量が製品の価値を決める
Module 05
コンポーネント設計
ピルはアクションの文法。形状が意味を持つ
Module 06
エレベーションと奥行き
UIに影を使わない。Frosted Glassが奥行きを作る
Module 07
レスポンシブ設計
縮小ではなく再設計。8つのブレークポイントの言語
Module 08
デザイントークン実装
Global → Alias → Component。1変更がシステム全体に伝播
Module 09
モーションとアニメーション
ease-out・0.6秒上限。気づかれないモーションが最高
Module 10
アクセシビリティ
コントラスト比4.5:1。すべての人が使えることが完成
Module 11
コンテンツ設計
20字以内の断言。テキストを削るほど製品が前に出る
Module 12
総合プロジェクト
すべてを統合。実際のプロダクトページを設計・実装する
