Module 03 · タイポグラフィ

タイポグラフィ体系

Appleのタイポグラフィは「静かに語る」。56pxのヒーロー見出しから10pxのマイクロリーガルまで、すべてのサイズに理由があり、すべてのウェイトに役割がある。

このモジュールで学ぶこと

フォントサイズを選ぶとき、多くの設計者は「見た目で判断する」。しかしAppleのタイポグラフィには判断のフレームワークがある。表示サイズにはネガティブトラッキング、ボディには17px、ウェイトは300/400/600/700のみ。これらのルールが組み合わさって、「Apple Tight」と呼ばれる独特のヘッドライン感が生まれる。

このモジュールの目標

タイプスケールのすべてのトークンを暗記するより、「なぜAppleはこのサイズとウェイトを選んだのか」を文章で説明できるようになること。

フォントファミリー

AppleはすべてのWebプロパティにプロプライエタリフォント SF Pro を使用している。SF Pro Display（19px以上向け）とSF Pro Text（それ以下向け）の2バリアントがあり、用途で使い分ける。

SF Pro Display
Beautiful design.
表示サイズ (19px+) · weight 600 · tracking -0.28px
SF Pro Text
Appleは製品のデザインと同じ真剣さでタイポグラフィに向き合う。17pxのボディテキストは、「読む」ための設計だ。
本文サイズ (17px) · weight 400 · tracking -0.374px · lh 1.47

代替フォント：Inter

SF Pro はAppleのシステムフォントであり、非Apple環境では自動的に代替フォントに落ちる。オープンソース代替として Inter（Google Fonts）が最も近い選択肢だ。

Interで近似するための調整

CSS スタック: Inter, system-ui, -apple-system, BlinkMacSystemFont, sans-serif — macOS/iOS/Safariでは実際のSF Proに解決される

表示サイズのletter-spacing: -0.01em を追加 — InterのデフォルトトラッキングはSF Proより広いため

ボディのline-height: 1.47 → 1.44 に微調整 — InterはSF Proよりx-heightが高いため、行間を少し詰める

weight 600 + font-feature-settings: "ss03" で、SF Proの丸みを帯びた「a」文字を近似

ネガティブトラッキング — Apple Tightの秘密

Appleのヘッドラインが「締まって見える」最大の理由は、ネガティブレタースペーシングだ。17px以上のすべてのテキストに、-0.12px から -0.374px のマイナストラッキングが適用されている。

✕ トラッキング 0（デフォルト）
Designed by Apple
letter-spacing: 0
✓ ネガティブトラッキング
Designed by Apple
letter-spacing: -0.374px

差は微妙に見えるが、長い見出しや複数行にわたるヒーロー見出しでは、印象が大きく変わる。Interを使う場合は、SF ProよりトラッキングがデフォルトでやYや広いため、さらに -0.01em を追加する。

ルール

ネガティブレタースペーシングは17px以上に限る。12px以下でネガティブトラッキングを適用すると可読性が著しく低下するため、小サイズには適用しない。これはAppleが実際に守っているルールだ。

ウェイト体系 — 500が存在しない理由

Appleのフォントウェイトは 300 / 400 / 600 / 700 の4段階だ。多くの設計者が使う500（Medium）は意図的に省かれている。

300 · Light Beautiful design.
400 · Regular Beautiful design.
500 · Medium Beautiful design.
600 · SemiBold Beautiful design.
700 · Bold Beautiful design.

500が存在しない理由は、コントラストの明確さだ。400と600の差は視覚的にはっきりしている。400と500の差は曖昧で、デザインの意図が伝わりにくい。Appleは「曖昧な中間を作らない」という姿勢を、ウェイトの選択にも適用している。

各ウェイトの役割

300 — Light ストアヒーローCTA (18px)、エンバイロメントリード (24px)。「軽やかな空気感」が必要な場面のみ使用。
400 — Regular デフォルト段落・タグライン (28px)・ナビリンク・キャプション。すべての「読まれるテキスト」の標準ウェイト。
600 — SemiBold すべてのヘッドライン・インライン強調・ボタンラベル。Appleの「見出しウェイト」はBold (700)ではなくSemiBold (600)。
700 — Bold タグライン (21px) で、より強い主張が必要なとき限定。スパリングに使用。

見出しは600。
700はタグラインがさらに主張したいときだけ。
500は存在しない。

ラインハイトの設計

Appleのラインハイトは「文脈ごとに最適化」されている。ディスプレイサイズは引き締め、ボディは余裕を持たせ、フッターリンクは驚くほど広く取る。

ディスプレイ
Big headline text that wraps
lh: 1.07
タイトに詰めて視覚的な塊を作る
ボディ
Appleの本文テキストは17pxで、行間は1.47。読書体験を設計している。
lh: 1.47
「読む」ための余裕を確保
フッターリンク
プライバシーポリシー
利用規約
サイトマップ
lh: 2.41
密なリスト列を「スキャン可能」にする

フッターのラインハイト 2.41 は一見「バグ」に見えるほど広い。しかしAppleのフッターは4〜5列のリンクリストが並ぶ情報密度の高い領域だ。ここだけ行間を大きく取ることで、各リンクがタップターゲットとして明確に分離される。

記憶すべき3つの数字

ディスプレイ: 1.07〜1.19（引き締め）・ボディ: 1.47（読書）・フッターリンク: 2.41（スキャン）。この3つが文脈を決定づける。

なぜボディは17pxなのか

現代のWebデザインにおけるボディテキストのデファクトスタンダードは16pxだ。Bootstrapも、Tailwindも、多くのデザインシステムも16pxをベースにしている。Appleは1px大きい17pxを選んでいる。なぜか。

16px（一般的なSaaS）

iPhone 17 Proは、これまでで最も高度なカメラシステムを搭載しています。プロ映像撮影機能が、あなたの手のひらに。

17px（Apple）

iPhone 17 Proは、これまでで最も高度なカメラシステムを搭載しています。プロ映像撮影機能が、あなたの手のひらに。

1pxの差は小さいが、大きなフォントサイズのページ全体に適用されると、「スキャンするサイト」から「読むサイト」への変化をもたらす。Appleのページは製品の価値を「説明する」ページだ。ユーザーに速読させるより、ゆっくり読んでもらう方がプロダクトの価値が伝わる。17pxはその哲学の具体的な表れだ。

16pxはスキャンのサイズ。
17pxは読書のサイズ。
Appleは読んでほしい。

演習

製品タイルをタイプセットする

Figma で新しいフレーム（1440×900px）を作成し、Apple的なタイポグラフィシステムを使って製品タイルを1枚タイプセットしてください。ビジュアル（画像）は使わず、テキストとカラーだけで構成します。

  1. 製品名を {typography.display-lg}（40px/600/0/lh1.10）でセット。letter-spacing を 0 に設定する（このサイズではAppleはトラッキングを0にしている）。
  2. タグラインを {typography.lead}（28px/400/+0.196px/lh1.14）でセット。プレビューして「軽さ」と「支持感」のバランスを感じる。
  3. 2つのCTAを並べる。1つは solid blue pill、1つは ghost blue pill。両方 {typography.body}（17px/400）でラベルをセット。
  4. 同じタイルをダークバージョンで複製する。背景を #272729 に変え、見出しと本文を白にする。リンクカラーを #2997ff に切り替えるべき箇所を特定する。
  5. タイルをライト・ダーク交互に4枚並べ、「タイルリズム」が実際に感じられるかを確認する。
