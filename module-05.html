Module 05 · コンポーネント

コンポーネント設計

Appleのコンポーネントはすべて「役割が1つ」だ。ナビゲーションはナビゲーション、ボタンはボタン。役割を越えない設計が、システム全体の一貫性を保つ。

このモジュールで学ぶこと

design.mdに定義されているAppleのコンポーネントは15種以上ある。しかしそれを「覚える」のがこのモジュールの目的ではない。各コンポーネントの構造・トークン参照・状態を読み解く力を養うことが目的だ。

Appleのコンポーネントドキュメントの特徴は「Default と Active/Pressed 状態のみ記録する」ことだ。Hover状態は記録しない。なぜなら、ホバーはモバイルに存在しないインタラクションだからだ。

このモジュールの目標

任意のコンポーネントを見たとき、「どのトークンで構成されているか」「どの状態があるか」を素早く分解できるようになること。

コンポーネントの解剖学

すべてのコンポーネントは以下の4層で記述できる。この枠組みを使うと、Appleのどのコンポーネントも同じ方法で分析できる。

層内容例（button-primary）
サーフェス背景色・形状・サイズbg: {colors.primary} · rounded: {rounded.pill} · padding: 11px×22px
コンテンツテキスト・アイコンのトークンcolor: white · {typography.body} (17px/400)
ボーダー線・アウトライン通常: なし · focus: 2px solid {colors.primary-focus}
状態Default / Active のみactive: transform scale(0.95)

Hover 状態はドキュメントしない。
Default と Active だけが、Appleのコンポーネント記録の原則だ。

ボタン文法の要点

ピル（9999px）= アクション。角丸スモール（8px）= ユーティリティ。角丸ミディアム（11px）= カード内セカンダリ。この3文法を混ぜない。

形状とボーダーラジウス

Appleのボーダーラジウスは役割と対応している。「なんとなく角丸」ではなく、値ごとに使用場面が決まっている。

{rounded.none} 0px 全面タイル。タイルは角丸なし。
{rounded.sm} 8px ユーティリティボタン・インライン画像
{rounded.md} 11px Pearl Buttonカプセル
{rounded.lg} 18px ストア・アクセサリカード
{rounded.pill} 9999px CTAピル・検索・チップ。アクションの文法。
{rounded.full} 50% 写真上の円形コントロールチップ

インプットとフォーム

Appleが分析されたページ上でドキュメントしているインプットは検索フィールドのみだ。バリデーションやエラー状態は記録されていない。

検索

bg: white · border: 1px rgba(0,0,0,0.08) · radius: pill · padding: 12×20px · height: 44px

注意点

検索インプットも {rounded.pill} を使う。ピル形状はCTAだけでなく「入力フィールドがアクティブなUI要素であること」を示す文法として使われる。

演習

コンポーネントを分解してドキュメントする

apple.comを開き、任意のページから3つのコンポーネントを選んで、「4層の解剖学フレーム（サーフェス・コンテンツ・ボーダー・状態）」でドキュメントしてください。

  1. デベロッパーツールで対象コンポーネントのCSSを確認し、background・border-radius・font-size・colorを記録する。
  2. design.mdのトークン定義と照合し、どのトークン名に対応するかを特定する（例: border-radius: 18px → {rounded.lg}）。
  3. Active/Press状態をトリガーし（スマホで実際に長押しするか、DevToolsでactive擬似クラスを適用）、変化を記録する。
  4. Figmaで同じコンポーネントをゼロから再現する。正確なトークン値を使い、Auto-layoutで構成する。
  5. 「なぜこのradiusが選ばれたのか」を1文で説明してコミュニティに投稿する。
