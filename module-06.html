Module 06 · エレベーション

エレベーションと奥行き

Appleは影を恐れるほど使わない。奥行きを作る手段は影ではなくカラーと素材だ——その哲学を徹底的に理解する。

「影=奥行き」ではない

多くのデザインシステムが「高い要素ほど強い影を持つ」という原則を採用する（Material Design など）。しかしAppleのアプローチはまったく逆だ。

apple.comで使われる影は1種類だけ。製品レンダリング（3DCGや写真）に現実世界の光源をシミュレートするためだけに使う。UIの階層を伝えるために影を使う場面は存在しない。

核心原則

UIコンポーネントに shadow を使うな。奥行きはサーフェスカラーの変化・半透明・ヘアラインの3手段で表現せよ。

エレベーション4段階

design.mdはエレベーションを4つのレベルで整理している。

Level 0 — Flat
shadow: none
border: none
全面タイル・フッター・グローバルナビ
border-radiusゼロが多い。ページの床面。
Level 1 — Hairline
border: 1px solid #e0e0e0
Utility Card・テーブル・セパレータ
影なしで「浮いているように見える」最小手段。
Level 2 — Frosted
backdrop-filter: saturate(180%) blur(20px)
bg: parchment @ 80%
Sub Nav・Floating Sticky Bar
「ガラス越しに後ろが見える」素材効果で浮遊感。
Level 3 — Product Shadow
box-shadow: rgba(0,0,0,0.22)
3px 5px 30px 0
製品PNG/レンダリングのみ
製品の物理的存在感を演出。UIには絶対に使わない。

Level 0 — フラット

Appleの全面タイルは完全にフラットだ。ナビゲーション・フッター・コンテンツタイルはすべて影もボーダーも持たない。

なぜ浮いて見えるのか？答えは背景色の差異だけ。白いセクションの隣にグレー（{colors.canvas-parchment}）または黒（{colors.surface-tile-1}）のセクションが来ることで、人間の視覚系が自動的に奥行きを認識する。

白 — canvas #ffffff
グレー — parchment #f5f5f7
ダーク — tile-1 #272729
白 — canvas #ffffff（再び）

影ゼロ。しかし目は奥行きを感じる。これがAppleのエレベーション哲学の核心だ。

Level 1 — ヘアラインとボーダー

1px のボーダーは影よりはるかに繊細だ。UIが浮いていることを主張しすぎず、ただ「ここに境界がある」ことを告げる。

ボーダーは常に {colors.hairline} (#e0e0e0) の1px。より薄い線（rgba）や2px以上の太い線は設計仕様に存在しない。

サブナビのヘアライン

Sub Navの下境界は rgba(0,0,0,0.08) という半透明のヘアラインだ。frostedガラスの上に乗ることを考慮した、透過性のある境界線だ。

Level 2 — Frosted Glass（磨りガラス）

Appleの最も特徴的なエレベーション表現がFrosted Glassだ。スクロール時も半透明のパネルが浮遊しているように見え、コンテンツとの「距離感」が生まれる。

Sub Navigation（Frosted Glass）
購入する

下にコンテンツが透けて見えることで「ガラスが浮いている」感覚を生む

CSSの実装は正確に以下の値を使う。ぶれてはならない。

background: rgba(245, 245, 247, 0.85);
backdrop-filter: saturate(180%) blur(20px);
-webkit-backdrop-filter: saturate(180%) blur(20px);
border-bottom: 1px solid rgba(0, 0, 0, 0.08);

saturate(180%) が重要だ。単純なblurではなく、背景の彩度を高めることでガラスの素材感が増す。この値を省略してはならない。

Level 3 — プロダクトシャドウ

design.mdに記録されている唯一の box-shadow 値は {elevation.product-shadow} だ。

影なし（誤）
製品が「ページに張り付いて」いるように見える
汎用shadow（誤）
Material Design風。Appleの仕様に存在しない。
Product Shadow（正）
rgba(0,0,0,0.22)
3px 5px 30px 0

正しい値の特徴を分析すると：

  • X offset 3px — わずかに右。自然光（左上から）の模倣。
  • Y offset 5px — 小さな縦オフセット。浮遊感よりも「台の上に置いてある感」。
  • blur 30px — 大きくソフトなぼかし。スタジオ照明を模倣。
  • spread 0 — 拡散なし。主張しすぎない。
  • opacity 0.22 — 22%という低不透明度が上品さを保つ。

このshadowはUIコンポーネントには
一切使わない。製品ビジュアルだけだ。

カードに影を付けたくなったら、それはヘアラインで解決せよ。

カラーによるエレベーション

UIの階層を影ではなくカラーで表現する方法を、改めて整理する。

カラートークン用途・意味
#ffffff {colors.canvas} ベース面。ページの最下層。
#f5f5f7 {colors.canvas-parchment} 第2層。features・testimonials等のセクション。わずかに持ち上がった印象。
#fafafc {colors.surface-pearl} Pearl Button背景。white より1段上の「磨かれた白」。
#272729 {colors.surface-tile-1} ダークサーフェス。white との強いコントラストで奥行きを作る。
#000000 {colors.surface-black} グローバルナビ。最も高い「権威」を持つ面。

この5色のサーフェスカラーが、影を一切使わずに5段階の視覚的奥行きを作り出している。

演習

エレベーション監査

あなたの手元のデザイン（Figmaファイル、個人プロジェクト、あるいは任意のWebサイト）のエレベーション使用状況を監査してください。

  1. DevToolsまたはFigmaで、box-shadow が使われているすべての要素をリストアップする。
  2. それぞれの shadow が「製品ビジュアルの現実感演出」なのか「UIの階層表現」なのかを分類する。
  3. UIの階層表現に使われている shadow をすべてヘアライン（1px solid #e0e0e0）に置き換えてみる。見た目はどう変わるか？
  4. Frosted Glass が使えるコンテキスト（スクロールで後ろにコンテンツが流れる固定パネル）を1箇所見つけて実装する。saturate(180%) を忘れずに。
  5. 置き換え前後のスクリーンショットを並べ、「どちらがAppleらしいか」をコミュニティに投稿する。
