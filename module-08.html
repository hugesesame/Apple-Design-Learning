Module 08 · トークン

デザイントークン実装

デザイントークンはデザインとコードの共通言語だ。design.mdのすべての値は実装可能なトークンとして設計されている——その実装方法を学ぶ。

なぜトークンが必要か

デザイナーが「青」と言い、エンジニアが #0066cc と書く。この2つが同じ意味を指しているとき、両者が「確認した」という記録はどこにも残らない。

デザイントークンはこの問題を解決する。{colors.primary} という名前が「デザイナーが青と呼んでいるもの」「エンジニアが #0066cc と書くもの」「Figmaでblue-primaryと呼ぶもの」をすべて同一のものとして接続する。

トークンは値ではなく意図に名前を付ける。
#0066cc に名前を付けるのではなく、
「プライマリアクションに使う青」に名前を付ける。

design.md Token Philosophy より

トークン3層構造

design.mdのトークンシステムは3層で構成される。この階層を理解せずにトークンを設計すると、必ず行き詰まる。

Tier 1 — Global Tokens（プリミティブ）
設計システムで使える全ての生の値を定義する。意味を持たない。
blue-600: #0066cc
size-17: 17px
↓
Tier 2 — Alias Tokens（セマンティック）
意図・用途を名前として持つ。Global Tokenを参照する。
color-primary: {blue-600}
font-body: {size-17}
↓
Tier 3 — Component Tokens（コンポーネント固有）
コンポーネントの特定のパーツに対してAlias Tokenを参照する。
button-primary-bg: {color-primary}
button-primary-text: {color-on-primary}

なぜ3層が必要か

ブランドカラーが #0066cc から #0052a3 に変わった場合、Global Tokenの blue-600 だけを更新すれば、それを参照するすべてのAlias・Component Tokenが自動的に更新される。1箇所の変更がシステム全体に伝播する。

命名規則

design.mdのトークン命名は {カテゴリ.用途-修飾子} のパターンに従う。

colors. カテゴリ接頭辞。colors / typography / spacing / rounded / elevation / component primary 意図または用途。何のための値か。 -on-dark （任意）コンテキスト修飾子。特定の背景上での使用を示す。

例: {colors.primary-on-dark} = 「ダーク背景でのプライマリカラー」

実際のトークン例

トークン値用途
{colors.primary} #0066cc 全ページ共通のプライマリアクション色
{colors.primary-on-dark} #2997ff 暗背景上での青リンク（アクセシビリティ調整）
{colors.canvas-parchment} #f5f5f7 パーチメントセクション背景

{elevation.product-shadow} rgba(0,0,0,0.22) 3px 5px 30px 0 製品レンダリング専用の唯一のshadow

CSS カスタムプロパティ実装

design.mdのトークンをCSSに実装するとき、:root セレクタにカスタムプロパティとして定義する。これがTier1〜2のトークンだ。

/* Tier 1 — Global Tokens */
:root {
  /* Blue scale */
  --blue-600: #0066cc;
  --blue-500: #0071e3;
  --blue-400: #2997ff;
  
  /* Neutral scale */
  --gray-900: #1d1d1f;
  --gray-200: #f5f5f7;
  --white: #ffffff;
}

/* Tier 2 — Alias Tokens */
:root {
  --color-primary: var(--blue-600);
  --color-primary-focus: var(--blue-500);
  --color-primary-on-dark: var(--blue-400);
  --color-ink: var(--gray-900);
  --color-canvas: var(--white);
  --color-canvas-parchment: var(--gray-200);
}

/* Tier 3 — Component Token（インライン例）*/
.button-primary {
  background: var(--color-primary);
  color: #ffffff;
}

CSS変数の二段階参照

Alias TokenがGlobal Tokenを参照する（var(--blue-600)）。コンポーネントがAlias Tokenを参照する（var(--color-primary)）。この二重の間接参照が、変更の一元管理を可能にする。

Figma Variables との対応

Figma Variables（2023年以降）は、design.mdのトークン構造と1:1で対応するように設計できる。

Figma Variables 構成
Global コレクション — blue-600, gray-900 など生の値
Semantic コレクション — color-primary → {blue-600}
Component コレクション — button-bg → {color-primary}

Figma Variables のモード機能を使うと、同じ変数名でライトモードとダークモードの値を切り替えられる。

/* Figma変数エクスポート → CSS変数への変換 */
/* ツール: Style Dictionary, Theo, Token Pipeline */

/* tokens.json (Design Token format) */
{
  "colors": {
    "primary": {
      "value": "#0066cc",
      "type": "color"
    }
  }
}

ライト/ダークモード

CSS変数を使ったモード切り替えは、Alias Tokenレイヤーで実現する。Global Tokenは変わらず、Alias Tokenの参照先だけが切り替わる。

ライトモード
詳しく見る
color-primary: #0066cc
color-ink: #1d1d1f
canvas: #ffffff
ダークモード
詳しく見る
color-primary: #2997ff
color-ink: #f5f5f7
canvas: #1c1c1e

ダークモードでプライマリカラーが青から明るい青（#2997ff）に変わる理由はコントラスト比の確保だ。#0066cc は暗背景では WCAG AAに届かない場合があるため、より明るい値が必要になる。

/* ライトモード（デフォルト）*/
:root {
  --color-primary: #0066cc;
  --color-ink: #1d1d1f;
  --color-canvas: #ffffff;
}

/* ダークモード */
@media (prefers-color-scheme: dark) {
  :root {
    --color-primary: #2997ff;
    --color-ink: #f5f5f7;
    --color-canvas: #1c1c1e;
  }
}

演習

トークンシステムを実装する

design.mdをベースに、CSS変数とFigma Variables の両方でトークンシステムをゼロから実装してください。

  1. 空のCSSファイルに :root ブロックを作成し、design.mdのカラートークンを全てCSS変数として定義する。Global → Alias の2段階で定義すること。
  2. そのCSS変数を使って、button-primary コンポーネントを実装する。background / color / border-radius / padding / font-size の5つを全てvar()で参照する。
  3. Figmaで新しいVariables コレクションを3つ作成（Global / Semantic / Component）し、同じ階層を再現する。
  4. @media (prefers-color-scheme: dark) を追加し、3〜5個のトークンをダークモード用の値でオーバーライドする。ブラウザのダークモード設定で動作を確認する。
  5. 「トークンがなかった場合、どこが壊れていたか」を説明するコメントをCSSに追加してコミュニティに投稿する。
