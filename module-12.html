Module 12 · 修了

総合プロジェクト

12のモジュールで学んだすべてを統合し、Appleデザイン言語に準拠した実際のプロダクトページを設計・実装する。これが最終課題だ。

最終モジュールへようこそ

あなたはここまで、Appleのデザインシステムを構成する12の領域を学んできた。哲学・カラー・タイポグラフィ・スペーシング・コンポーネント・エレベーション・レスポンシブ・トークン・モーション・アクセシビリティ・コンテンツ——これらすべては独立した知識ではなく、互いを支え合うシステムだ。

最終モジュールの目的は、この知識を「知っている」から「使える」へ変換することだ。アウトプットなき学習は知識のストック。アウトプットがあって初めてスキルになる。

学んだことは、作ることで初めて
あなたのものになる。

Module 12 — Final Project

12モジュールの核心を振り返る

各モジュールから「絶対に忘れてはならない1つの原則」を抽出する。

MODULE 01 · 哲学
Less, but better
アクセントカラーは1色。影は1種類。フォントウェイトは4段階。「削ること」がAppleのデザイン行為だ。
MODULE 02 · カラー
セマンティックカラー
#0066ccを直接使わない。{colors.primary}を使う。意図に名前を付けることで、変更が伝播する。
MODULE 03 · タイポグラフィ
17pxは設計値
ボディテキストは17px（16pxではない）。Appleの読書ペースへの配慮だ。letter-spacingは必ず設定する。
MODULE 04 · スペーシング
80pxの台座
セクション垂直パディング80pxは「製品を美術館の台座に乗せる」行為だ。余白の量が製品の価値を示す。
MODULE 05 · コンポーネント
ピルはアクションの文法
border-radius: 9999pxはCTAとアクション要素のみ。角丸8pxはユーティリティ。この文法を混ぜない。
MODULE 06 · エレベーション
UIに影を使わない
box-shadowは製品レンダリング専用。UIの階層はカラー変化・ヘアライン・backdrop-filterで表現する。
MODULE 07 · レスポンシブ
縮小ではなく再設計
モバイルはデスクトップのサブセットではない。8つのブレークポイントで、それぞれに最適な体験を作る。
MODULE 08 · トークン
意図に名前を付ける
Global → Alias → Componentの3層。1箇所の変更でシステム全体を更新できる構造がトークンの目的だ。
MODULE 09 · モーション
ease-outと0.6秒上限
Appleのアニメーションはease-outで短く終わる。装飾でなく情報の遷移を助ける機能的モーションだ。
MODULE 10 · アクセシビリティ
コントラスト比4.5:1
WCAG AA達成は全ユーザーへの責任。:focus-visible・セマンティックHTML・reduced-motionは必須実装だ。
MODULE 11 · コンテンツ
20字以内の断言
ヒーローコピーは主語を省いた断言。CTAは3語以内。テキストを削るほど製品が前に出る。
MODULE 12 · 統合
すべては一つのシステム
11の原則はバラバラに機能しない。互いが条件となり支え合うとき、「Appleらしさ」が生まれる。

最終提出チェックリスト

プロジェクトを提出する前に、以下のすべてを確認する。1つでも未達成の項目があれば、そこがまだ学習中の領域だ。

MODULE 01 · 哲学
アクセントカラーは1色のみ
{colors.primary}（#0066cc）以外の色でCTAやリンクが着色されていないか確認する。
MODULE 02 · カラー
すべてのカラーがトークン参照
CSSに直接 #0066cc と書いていない。var(--color-primary) で参照している。
MODULE 03 · タイポグラフィ
ボディ17px / letter-spacing設定済み
font-size: 17px。letter-spacing: -0.374px（ボディ）。font-weight: 300/400/600/700のみ。
MODULE 04 · スペーシング
セクションパディング80px以上
各セクションの上下パディングが{spacing.section}=80pxになっているか確認する。
MODULE 05 · コンポーネント
ボタンの形状文法が正しい
CTAはpill(9999px)。ユーティリティは8px。カードセカンダリは11px。混在していない。
MODULE 06 · エレベーション
UIコンポーネントにbox-shadowなし
カード・ボタン・ナビに box-shadow が使われていない。製品ビジュアルのみに限定されている。
MODULE 07 · レスポンシブ
833px / 419pxで崩れない
DevToolsで両ブレークポイントを確認。テキスト重複・はみ出し・タッチターゲット44px未満がない。
MODULE 08 · トークン
CSS変数が3層で定義されている
:rootにGlobal・Aliasトークンが定義されており、コンポーネントはAliasを参照している。
MODULE 09 · モーション
ボタンactive: scale(0.95)実装済み
button:active { transform: scale(0.95); } が 150ms ease-out で実装されている。
MODULE 09 · モーション
prefers-reduced-motion対応済み
@media (prefers-reduced-motion: reduce) でアニメーションが無効化される。
MODULE 10 · アクセシビリティ
Lighthouse アクセシビリティ90+
ChromeのLighthouseでアクセシビリティスコア90点以上を達成している。
MODULE 11 · コンテンツ
ヒーローコピーが20字以内の断言
Lorem ipsumが残っていない。CTAが「今すぐ」などの煽りワードを含まない。

総合プロジェクト — 仕様

実在または架空のプロダクト1点の製品ランディングページを、Figma（デザイン）とHTML/CSS（実装）の両方で作成する。

必須セクション

  • Global Nav — 44px、黒背景、スティッキー
  • Sub Nav — 52px、Frosted Glass、製品名＋CTA
  • Hero タイル — ダーク背景、20字以内のタイトル、2CTAボタン
  • Features タイル — パーチメント背景、3つ以上の機能説明
  • Product タイル — ライト背景、製品ビジュアル（product shadow）
  • フッター — 5カラム、全フルwidthブラック

技術要件

  • CSS変数（3層トークン構造）を使用する
  • 833px と 419px のブレークポイントで適切に表示される
  • linear-gradient をプロダクトモックアップに使用しない
  • UIコンポーネントに box-shadow を使用しない
  • prefers-reduced-motion に対応する
  • Lighthouse アクセシビリティスコア 90 以上

Figmaの提出も必須

HTMLの実装だけでなく、Figmaファイルも提出する。Variables（3層）を設定し、コンポーネントライブラリとして整理すること。デザインとコードが一致していることを確認する。

この先へ

design.mdはAppleのデザイン言語の入り口だ。design.mdが記録しているのはapple.comの表層パターンであり、Appleが実際に使っているプロセスや意思決定の詳細は公開されていない。

ここから先の学習パスとして、以下を推奨する。

Apple Human Interface Guidelines
developer.apple.com — iOS/macOSのUI設計原則。apple.comとは別の設計言語を持つが、哲学は共通している。
実際のapple.comを定期的に監査する
Appleは月単位でデザインを更新する。DevToolsで新しい値を発見し、design.mdとの差分を記録し続けることが上達の道だ。
自分のデザインシステムを作る
学んだトークン設計・コンポーネント解剖学・エレベーション哲学を、自分のプロジェクトに適用したオリジナルのシステムを構築する。
